## `1. 개요 (Overview)`
> 프로젝트의 정의와 목표를 간략히 기술합니다.
- **프로젝트명**: AI TechTree
- **한 줄 소개**: AI 에이전트 면접관과 함께 성장하는 RPG형 개발자 로드맵
- **핵심 목표**: 단순한 기술 점검을 넘어, **AI 에이전트와의 심층 인터뷰**를 통해 사용자의 기술 숙련도(1차~3차 전직)를 진단하고, 트랙 완주 시 시각적 보상(Gold Glow)을 제공하여 성장의 즐거움을 부여한다.

## 1.1 MVP 목표 및 범위 (MVP Goals & Scope)
> 최소 기능 제품(MVP)에서 검증하고자 하는 핵심 가치와 개발 범위를 정의합니다.
- **MVP 핵심 가치**: 
    - "나의 개발 실력을 **별(Star)** 개수로 증명할 수 있는가?"
    - "AI 면접관이 내 답변을 이해하고 적절한 **승급 심사(전직)**를 진행하는가?"
- **포함 기능 (In-Scope)**:
    - **초기 역량 배치고사** (가입 시 1차 전직 일괄 부여)
    - **레벨별 승급 심사 에이전트** (1차/2차/3차 전직 면접)
    - **트랙 마스터 통합 퀴즈** (Boss Challenge)
    - **Tech Tree 시각화** (별 시스템 & 골든 글로우 이펙트)
- **제외 기능 (Out-of-Scope)**:
    - 트렌드 스카우터 에이전트 (Phase 2 예정)
    - 커리어 코칭 및 강의 추천 (Phase 2 예정)
    - 유저 간 랭킹 경쟁 및 소셜 기능

## `2. 유저 스토리 (User Stories)`
> 유저가 서비스를 사용하며 얻고자 하는 가치를 시나리오 형태로 작성합니다.
- **취업 준비생**은 자신의 Python 실력이 단순히 '아는 정도'인지 '응용 가능한지' 확인하기 위해 **2차 전직 심사에 도전하여 별 2개를 획득**하고 싶어한다.
- **현업 개발자**는 MLOps 트랙의 모든 기술을 마스터했음을 증명하기 위해 **통합 퀴즈(Boss)를 통과하고 트랙을 황금색으로 빛내어** 포트폴리오에 첨부하고 싶어한다.
- **비전공자**는 어떤 기술부터 배워야 할지 모르므로, **활성화된 1차 전직 노드**를 따라가며 게임 퀘스트를 깨듯이 자연스럽게 학습하고 싶어한다.

## `3. 핵심 기능 (Key Features)`
> 서비스가 제공해야 할 구체적인 기능들을 나열합니다.
### 3.1. 초기 역량 배치고사 (Initial Skill Calibration)
- **기능 설명**: 설문조사 및 간단한 종합 퀴즈를 통해 여러 기술 스택의 **1차 전직(⭐)** 상태를 한 번에 부여.
- **목적**: 모든 기술에 대해 일일이 1차 심사를 받는 피로감을 줄이고, 사용자가 빠르게 자신의 Tech Tree를 형성하도록 도움.
- **제약 사항**: 이 테스트로는 **최대 1차 전직**까지만 획득 가능. (2차/3차는 반드시 개별 에이전트 심사 필요)
- **입력 데이터**: 사용자 직무(예: 백엔드), 경력 연차, 주요 사용 기술 체크리스트.
- **주요 로직**:
    - 사용자가 "Python, Docker 사용 가능" 체크 -> 해당 기술들에 대한 기초 퀴즈(2-3문제)를 출제.
    - 정답률 80% 이상 시 해당 노드들 일괄 **1차 전직(⭐)** 처리.

### 3.2. 승급 심사 에이전트 (Advancement Interview Agent)
- **기능 설명**: 각 기술 노드(예: Docker)의 숙련도를 1~3단계(전직)로 평가하는 AI 면접관.
    - **1차 전직 (⭐)**: "기초 개념 확인" (단답형 위주)
    - **2차 전직 (⭐⭐)**: "응용 및 구현 능력" (코드 스니펫 설명, 트러블슈팅 시나리오)
    - **3차 전직 (⭐⭐⭐)**: "심화 및 최적화" (아키텍처 설계, 성능 튜닝)
- **입력 데이터**: 도전하려는 전직 등급, 사용자 답변(텍스트/음성).
- **출력 결과**: 합격/불합격 여부, 획득한 별(Star), 피드백 리포트.
- **주요 로직 (LangChain)**:
    - **Level-Adaptive Prompting**: 현재 도전 중인 등급(1~3차)에 맞춰 프롬프트의 난이도와 평가 기준(Rubric)을 동적으로 변경.
    - **Evaluation Agent**: 사용자의 답변이 기준치(예: 80점)를 넘으면 `pass` 플래그와 함께 DB 업데이트.

### 3.3. 트랙 마스터 통합 퀴즈 (Track Master Challenge)
- **기능 설명**: 특정 직무 트랙(예: AI Engineer 트랙)의 필수 노드를 일정 수준 이상 달성 시 도전 가능한 **최종 관문(Boss Battle)**.
- **입력 데이터**: 해당 트랙 내의 전 기술에 대한 종합적 질문.
- **출력 결과**: 트랙 마스터 자격 획득 여부 (Success/Fail).
- **주요 로직**:
    - **Scenario Generation**: "Docker로 배포된 FastAPI 서버가 메모리 누수를 일으킬 때 해결 방안을 제시하시오"와 같이 여러 기술(Docker + Python + FastAPI)이 융합된 복합 문제 출제.
    - **Strict Scoring**: 부분 점수 없이 엄격한 기준으로 합격 여부 판단.

### 3.4. Tech Tree 시각화 (Gamified Visualization)
- **기능 설명**: 사용자의 성장 상태를 게임 스킬 트리 형태로 시각화.
- **시각적 요소**:
    - **Nodes**: 기술 아이콘 + **획득한 별(Star ⭐⭐⭐)** 표시.
    - **Edges**: 연결된 노드가 모두 별이 **1개 이상** 획득된 상태일 때만 연결선이 활성화.
    - **Track Glow**: 트랙 마스터 합격 시, 해당 트랙의 모든 노드와 연결선이 **밝은 금색(Golden Glow)으로 발광**하는 애니메이션 효과 적용.
- **기술 스택**: `React Flow` (Custom Node 개발), `Framer Motion` (애니메이션).

## `4. 비기능 요구사항 (Non-Functional Requirements)`
> 성능, 보안, 사용성 등 기능 외적인 요구사항을 정의합니다.
- **사용성 (UX)**:
    - 전직 심사 중 AI의 답변 생성 딜레이가 느껴지지 않도록 **Streaming Response** 적용.
    - 획득한 별이 차오르거나 트랙이 빛나는 효과는 사용자의 성취감을 위해 화려하지만 성능을 저해하지 않도록 최적화
- **데이터 무결성**:
    - 2차 전직은 반드시 1차 전직(별 1개)을 획득한 상태에서만 도전 가능하도록 백엔드에서 검증 로직 수행.
- **확장성**:
    - 추후 '4차 전직(신/God 등급)'이나 '히든 트랙'을 추가할 수 있도록 DB 스키마를 유연하게 설계.

## `5. 데이터 요구사항 (Data Requirements)`
> 서비스 운영에 필요한 핵심 데이터 구조를 정의합니다.
- **UserSkill (Collection)**:
    - `user_id`: 사용자 ID
    - `skill_name`: 기술명 (ex: python)
    - `advancement_level`: 현재 전직 단계 (0, 1, 2, 3)
    - `is_track_master`: 해당 트랙 마스터 여부 (Boolean)
- **InterviewLog**: 면접 대화 내용 및 AI 평가 결과 저장 (추후 RAG 데이터로 활용 가능).

## `6. 성공 지표 (Success Metrics)`
> 프로젝트의 성공 여부를 판단할 기준입니다.
- **승급 도전율**: 1차 전직 완료 유저가 2차 전직 심사에 도전하는 비율.
- **트랙 완주 수**: 통합 퀴즈(Boss)를 통과하여 'Golden Glow'를 획득한 유저 수.
- **바이럴 지표**: 자신의 완성된 Tech Tree 이미지를 캡처하여 공유한 횟수.
